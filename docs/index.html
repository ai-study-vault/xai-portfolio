Three Techniques

Explainability in depth

LIME — Local Interpretable Model-agnostic Explanations

LIME explains individual predictions by perturbing the input features and fitting a simple linear model in the local neighbourhood. It answers: "Why did the model predict this for this specific patient?"

Local
  • Perturbs the input 1,000× and records how the prediction changes
  • Fits a weighted ridge regression — weights = similarity to original point
  • Green bars push toward Benign; red bars push toward Malignant
  • Works on any model (black-box agnostic) — classifiers, regressors, even deep nets
  • Limitation: explanations can be unstable across nearby instances
SHAP — SHapley Additive exPlanations

SHAP is grounded in cooperative game theory. It assigns each feature a Shapley value — its fair contribution to the prediction, averaged over all possible feature orderings. Produces both local and global explanations.

Local + Global
  • Beeswarm (left): each dot is one patient — colour = feature value, x-axis = SHAP impact
  • Bar chart (right): mean |SHAP| ranks features by global importance
  • TreeExplainer computes exact SHAP values in O(TLD²) — fast for tree models
  • Additivity: SHAP values sum to the gap between prediction and base rate
  • Satisfies three desirable axioms: efficiency, symmetry, and dummy-player
PDP — Partial Dependence Plots

PDPs show the marginalised effect of one feature on the predicted outcome, averaging over the distribution of all other features. They answer: "How does the model's average prediction change as I vary this feature?"

Global
  • For each grid point, substitute the feature value across all training rows and average the predictions
  • Reveals monotonic vs. non-monotonic relationships — does increasing X always raise P(Benign)?
  • Top 4 features selected by SHAP importance for consistency across methods
  • Shaded area shows 95% CI over the population distribution
  • Assumption: features are independent — can be misleading with correlated inputs (use ALE instead)
Method Comparison

Choosing the right tool

Method Scope Model-agnostic Theoretically grounded Handles interactions Best for
LIME Local ✓ ~ ✗ Explaining individual predictions to stakeholders
SHAP Local + Global ✓ ✓ ✓ Audits, feature selection, fairness analysis
PDP Global ✓ ~ ✗ * Communicating feature effects to non-technical audiences
ICE Global ✓ ~ ✓ Detecting heterogeneous effects (PDP extension)
ALE Global ✓ ✓ ~ Correlated features where PDP misleads

* 2D PDPs can show interactions between two features simultaneously.

Reproducible Code

Quick-start snippets

import lime.lime_tabular

# Build explainer once — reuse for any instance
explainer = lime.lime_tabular.LimeTabularExplainer(
    X_train,
    feature_names=feature_names,
    class_names=['Malignant', 'Benign'],
    mode='classification',
)

# Explain one test sample
exp = explainer.explain_instance(
    X_test[0],
    model.predict_proba,
    num_features=10,
)

# Render inline in a Jupyter notebook
exp.show_in_notebook(show_table=True)

# Or get raw weights
weights = exp.as_list()  # list of (feature_condition, weight)
import shap

# Fast exact SHAP for tree-based models
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_test)

# Global summary — beeswarm
shap.summary_plot(shap_values[1], X_test,
               feature_names=feature_names)

# Mean |SHAP| bar chart
shap.summary_plot(shap_values[1], X_test,
               feature_names=feature_names,
               plot_type='bar')

# Local waterfall for one prediction
shap.waterfall_plot(
    shap.Explanation(
        values=shap_values[1][0],
        base_values=explainer.expected_value[1],
        data=X_test[0],
        feature_names=feature_names,
    )
)
from sklearn.inspection import PartialDependenceDisplay
import matplotlib.pyplot as plt

# Pick top features by SHAP importance
top_features = [0, 2, 7, 20]

# 1-D PDPs for each feature
fig, ax = plt.subplots(figsize=(12, 4))
PartialDependenceDisplay.from_estimator(
    model,
    X_train,
    features=top_features,
    feature_names=feature_names,
    kind='average',   # 'individual' for ICE, 'both' for ICE+PDP
    ax=ax,
)
plt.tight_layout()
plt.show()

# 2-D interaction PDP
PartialDependenceDisplay.from_estimator(
    model, X_train,
    features=[(0, 2)],   # tuple = 2-D heatmap
    feature_names=feature_names,
)
